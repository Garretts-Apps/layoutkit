LayoutKit is a pure-CSS stylesheet. It styles semantic <lk-*> custom tags through plain attribute selectors — no JavaScript, no custom-element registration, no build step. Each tag below is a normal HTML element that the stylesheet applies layout rules to.

This page itself is built with LayoutKit: the nav, grids, and stacks you see are <lk-*> tags. Drop in one stylesheet and write tags. 

<link rel="stylesheet" href="https://unpkg.com/layoutkit-css/layoutkit.css">
/* or, with a bundler */
import "layoutkit-css/layoutkit.css";

<lk-center>

Center content on both axes. The #1 CSS pain point, solved — no JavaScript required.

 
<lk-center full-height>
  <form>Login</form>
</lk-center>

inlinebooleanoffUse inline-flex instead of flex

<lk-aspect-ratio>

Hold a fixed width-to-height ratio. Media embeds, image placeholders, cards.

 
<!-- 16:9 video frame -->
<lk-aspect-ratio style="--lk-ratio: 16 / 9">
  <iframe src="…" style="width: 100%; height: 100%"></iframe>
</lk-aspect-ratio>

Note: the ratio is enforced on the <lk-aspect-ratio> box itself. Its child must size itself to the box (e.g. width: 100%; height: 100% or object-fit: cover) — the tag does not stretch children for you.

ATTRIBUTES
AttributeTypeDefaultDescription
--lk-ratioratio (custom prop)1Width-to-height ratio, e.g. 16 / 9. Set inline via style.
APPLIED CSS
SelectorApplied CSS
lk-aspect-ratio (base)display: block; aspect-ratio: var(--lk-ratio, 1)

<lk-scroll-area>

A scrollable viewport with capped dimensions. Long lists, code panes, overflow regions.

 
<lk-scroll-area style="--lk-max-height: 320px">
  <ul> … long list … </ul>
</lk-scroll-area>

<!-- horizontal scroller -->
<lk-scroll-area direction="horizontal" style="--lk-max-width: 100%"> … </lk-scroll-area>
